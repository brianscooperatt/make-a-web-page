    What is HTML Markup?

  • HTML markup is made up of "tags" and "elements."
  • The tags and the content in between the tags makes up the ELEMENT.
  • Some common tags that go between < and > include:

    • 1. B tag (bold)
    2. EM tag (italics)
    3. A tag (anchor tag for making links)
    4. IMG tag (images)
    5. ALT tag (alternate text to more clearly define in words what an image is about - often used for broken requests or for the blind
    6. BR tag (break tag) which is an INLINE tag.
    7. P tag (paragraph tag) which is a BLOCK tag.

    ***Note***: INLINE and BLOCK tags form an invisible box around the text/content.

    What are HTML Attributes? - HTML attributes (attr) go inside of tags to change things such as font size, color, etc.

    HTML Vocabulary:

    TAG: HTML tags are contained within brackets. Most tags have an opening tag and a closing tag that ends with a slash.
    ELEMENT: Everyting within a set of opening and closing tags.
    ATTRIBUTE: Further defining an HTML element
    SPAN: Inline element
    DIV: Block element

    LESSON 2: Creating a Structured Document with HTML

    - A Website is like a House-The HTML is the structure of the home including the walls and frame, CSS is the style of the home including the color and layout, and Java Script is the interactive part of the home including remote controls, TV's, etc.'

    - Boxes Everywhere!- Everything you see on Websites can be broken down into boxes and within those boxes, the content can be changed using HTML, CSS and Java Script.

    - Developer Tools (in the Browser)- HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

    - The "tree-like structure" of HTML - The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. My mother had multiple children. So did her mother, and so on... In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

    - The relationship between indented HTML and boxes - When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.
